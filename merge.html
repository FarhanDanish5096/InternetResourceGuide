Characteristics: Merging Strategies

Merging Strategies

Git offers multiple merge strategies to handle various branch merging scenarios.

Key Strategies

  • Resolve: Traditional 3-way merge used for simple cases.
  • Recursive (default for two branches): A 3-way merge that can auto-resolve some conflicts using a merged base of multiple common ancestors.
  • Octopus (default for >2 branches): Used when merging multiple heads simultaneously, typically for combining many feature branches.
