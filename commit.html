Features: Commit

What is a Commit?

A commit is a snapshot of your repository at a specific point in time. It records changes made to one or more files, along with metadata such as the author, timestamp, and a unique identifier known as a SHA hash.

Key Details about Commits

  • Change tracking: Each commit logs what changes were made and by whom.
  • Commit messages: Provide context and reasoning behind changes, aiding in project history comprehension.
  • Unique identifiers: Every commit has a SHA hash, ensuring integrity and traceability.

Advanced Concepts

  • Co-authored commits: Multiple contributors can be credited in a single commit.
  • Rebasing: Allows you to modify the commit history for a cleaner project timeline.
  • Commit signoffs: Some repositories require commits to be signed off, indicating agreement to the project's contribution guidelines.
