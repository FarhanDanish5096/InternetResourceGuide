Commands: Init

Using Git Init

Starting a new git repository.

What happens?

  • Initializes a new Git repository in the current directory.
  • Creates a hidden .git directory where Git stores all internal data (like commits, branches, and configuration).
  • Does not create a remote repository â€” it's only local unless connected later.

When to use?

  • You're starting a new project and want to begin tracking its history with Git.
  • You cloned a repo and need to re-initialize it (rare, usually only for troubleshooting).

Useful Tip(s)

  • Run git status after init to check if the repository is tracking anything yet.
